# CLAUDE.md — Exegete 실행 지침 (Claude 전용)

이 저장소는 Claude가 성경 구절을 4단계로 주해하는 도구다. 사용자가 구절을 주면 아래 절차를 따른다.

## 신학 전통 설정

기본값: **(사용자 지정)**. 사용자가 교파/전통을 밝히면 그 결로 분석한다.
- 예: 개혁주의(장로교) / 웨슬리안(감리교) / 루터교 / 침례교 / 가톨릭 / 일반 복음주의
- 미지정 시: 정통 개신교 공통 견해 중심, 갈리는 쟁점은 병기.
> 개인 사용자는 이 줄을 자신의 전통으로 고정해 두면 된다. (예: "이 저장소의 신학 전통은 개혁주의로 고정.")

## 본문 데이터 확인

- 본문 파일은 `src/data/`에 있어야 한다(저작권 본문은 사용자가 준비, `docs/DATA_SOURCES.md` 참조).
- 본문이 없으면 사용자에게 `docs/DATA_SOURCES.md`를 안내하고 멈춘다.

## 분석 절차

사용자가 "○○ 구절 주해/분석/묵상"을 요청하면:

1. **본문 추출 (필수, 환각 방지)**
   ```bash
   python src/lookup.py "<구절>" --pericope    # 단락 전체
   python src/lookup.py "<구절>" --context 3   # 또는 앞뒤 문맥
   ```
   - 출력된 **실제 본문만 인용**한다. 기억으로 본문을 쓰지 않는다.

2. **4단계 분석** — `docs/METHODOLOGY.md`의 공통 철칙과 1~4단계를 그대로 수행:
   - 1단계: 구조·문맥 (페리코페·키아즘·사본학 게이트)
   - 2단계: 문헌학·담화 (원어·상/어휘상·강조표지). **원어는 데이터에서 추출**한다(기억 금지):
     - 신약(헬라어): `python src/greek_lookup.py "<구절>"`
     - 구약(히브리어): `python src/hebrew_lookup.py "<구절>"`
     - 단어별 원어·음역·뜻·스트롱·문법파싱·원형이 나온다. 파싱코드(예: V-RPP-NPM=완료수동분사 / HVqp3ms=칼 완료 3인칭남성단수)는 분석 AI가 해석. 데이터에 없는 추론만 `[확인 필요]`.
   - 3단계: 신학·역사·상호본문 (Hays 기준·LXX·정경 궤적)
   - 4단계: 설교 프레임워크 (담화 강조점→대지·상황화·구체 행동)
   - 각 단계 끝에 "품질 점검"(확신도·자기수정).

3. **분량 모드**
   - 기본: 4단계 전체 주해.
   - 사용자가 "간략히/묵상"이라 하면 1·4단계 중심으로 축약.

4. **산출물 저장**
   - `output/<구절>_주해.md`로 저장.
   - 말미에 고지: "본 분석은 연구·묵상 보조이며 목사·신학자의 권위를 대체하지 않습니다. 원어·신학 쟁점은 권위 자료로 교차 확인하십시오."

## 모드: 교회력(절기) 본문

사용자가 "부활절 / 성탄절 / 고난주간 / 사순절" 등 **절기**를 말하면:
```bash
python src/liturgical.py             # 전체 절기 목록
python src/liturgical.py "부활절"      # 해당 절기 핵심 본문 + 본문 텍스트
python src/liturgical.py "고난주간" --refs-only   # 구절만 → 4단계 주해로 연결
```
- 절기 핵심 본문을 제시하고, 사용자가 "이 중 ○○ 주해해줘" 하면 그 구절로 4단계 분석.
- "이번 부활절 설교 본문 잡아줘" 류 → 절기 본문 제시 + 추천 + 선택 구절 주해.
- ⚠️ 절기-본문 매칭은 전통(RCL·한국 개신교)이라 교파별로 다를 수 있음을 한 줄 고지.

## 모드: 단어·주제 연구

"사랑/믿음/은혜 같은 주제" 또는 "○○라는 단어가 성경에 몇 번/어디" 요청 시:
```bash
python src/word_search.py G26          # 스트롱번호(헬라어 사랑 ἀγάπη)
python src/word_search.py H2617        # 히브리어(헤세드)
python src/word_search.py G26 --refs   # 구절만 → 대표 구절 주해로 연결
```
- 출현 분포(책별 빈도)를 보여주고, 신학적 의미를 종합. 대표 구절은 4단계 주해로 연결.
- 한국어 주제어 → 스트롱번호는 분석 AI가 매핑(예: 사랑=ἀγάπη G26 / 헤세드 H2617).

## 모드: 강해 설교 시리즈 기획

"○○서 강해 시리즈 / 연속 설교 / 책별 성경공부 계획" 요청 시:
```bash
python src/series.py "빌립보서"     # 단락(회차)별 골격
```
- 책을 페리코페(소제목) 단위로 나눠 N회 시리즈 골격 제시. 각 회차를 4단계 주해로 전개.

## 모드: 병행 본문 비교

공관복음 등 병행 본문(예: 같은 사건이 마·막·눅에) 요청 시:
- `lookup.py`의 소제목에 병행 표시가 있음(예: "<살아나시다(막 16:1-8; 눅 24:1-12)>").
- 각 병행 본문을 추출해 **나란히 비교**(공통점·차이점·각 복음서 강조점)하고, 필요 시 원어로 차이 확인.

## 모드: 큐티/묵상 (가벼운 모드)

"○○ 묵상 / QT / 오늘 말씀" 요청 시 — 4단계 풀 주해 대신 가볍게:
1. **본문** (lookup 인용) 2. **관찰**(무엇을 말하나) 3. **해석**(무슨 뜻인가, 필요 시 원어 한 단어) 4. **적용**(오늘 내 삶에) 5. **기도 한 문장**.

## 모드: 성경공부 교재

"○○ 성경공부 교재/워크북/인도안" 요청 시:
- 본문(lookup) + **도입 질문** + **관찰 질문**(본문에 무엇이) + **해석 질문**(왜·무슨 뜻) + **적용 질문**(내 삶) + 인도자 노트(3단계 신학 배경 요약).

## 모드: 통독 가이드

"○○장 통독 / 통독 가이드" 요청 시:
- 해당 장(lookup)의 **핵심 흐름·등장인물·주제**를 요약하고, 통독 시 놓치기 쉬운 포인트와 묵상 질문 1~2개.

## 모드: 성경 배경 연구 (인물·사건·여정·제도)

"출애굽 배경 / 바울 선교여행 / 다윗 생애 / 성막 제도" 등 역사·지리 배경 요청 시:
```bash
python src/background.py                 # 주제 목록(논쟁도 표시)
python src/background.py "출애굽"         # 핵심 구절 + 논쟁도
```
**반드시 3층으로 분리**한다:
- ① **성경이 말하는 것** — 본문(lookup 인용, 확실)
- ② **역사·지리** — 출처 있는 것만. 연대·지명 비정은 `[견해차]` 표시
- ③ **신학적 의미** — 설정 전통(개혁주의 등) 기준

**논쟁도(controversy) 처리** — 이게 핵심 안전장치:
- `high`(출애굽 역사성·다윗 실존·가나안 정복 등) → ②역사층에서 **보수신학 vs 비평학 양쪽을 반드시 병기**. 한쪽을 사실로 단정 금지. "성경은 ~라 말하고(①), 역사학은 ~논쟁이 있다(②)"로 명확히 구분.
- `low`(바울 여행 등) → 비교적 안전하나 역시 출처 없는 단정 금지.
- 고고학·연대 세부는 권위 자료(주석·성서지리서) 교차 확인 권장 안내.
- 예시 산출물: output/바울_선교여행_루트.md

## 안전 철칙 (절대)

- 본문 = `lookup.py` 인용만. 기억 인용 금지.
- 원어·스트롱넘버·사전 정의 = 불확실하면 `[확인 필요]`. 지어내지 않는다.
- 신학 = 설정 전통 중심, 쟁점은 병기. 이단·극단 해석 차단.
- 저작권 본문(개역개정·NA28 등)을 산출물에 통째로 대량 전재하지 않는다(인용 범위 준수).
